Direction A

Immersive Full-Bleed

The hex grid fills the entire screen. Status and controls float as translucent overlays. Maximum map real estate. The app disappears; the battlefield remains.

Direction A Notes

  • Maximum hex grid real estate — status and controls are translucent overlays
  • The map truly IS the interface — nothing competes with it
  • Translucent bars ensure map content behind them is partially visible
  • Risk: overlapping content in dense late-game states
  • Best for: players who want full immersion and maximum map visibility
Direction B

Clean Segmented

Clear visual boundaries between status, map, and controls. Each zone is distinct. Predictable layout, easy to scan. The map is prominent but contained within defined bounds.

Direction B Notes

  • Clear visual separation between status, map, and controls
  • Turn summary strip sits between status bar and map — always visible
  • Journal slides up from bottom, pushing the map up but keeping it partially visible
  • Most predictable layout — each zone is always in the same place
  • Slightly less map space than Direction A, but no overlapping content
  • Best for: clarity-first design that prioritizes readability over immersion
Direction C

War Table

Atmospheric borders evoke a commander's tent. Warm wood-toned edges frame the map. The hex grid feels like it's drawn on parchment under lamplight. Maximum thematic immersion.

Direction C Notes

  • Warm wood-toned gradients at top and bottom create a "frame" around the map
  • Subtle vignette effect on the map edges adds depth and atmosphere
  • Status labels use Cinzel font for thematic consistency
  • Strongest thematic identity of all directions — most "fantasy" feeling
  • Looks best for OBS recordings — the atmospheric framing creates a polished visual
  • Risk: decorative elements may age poorly or feel heavy over time
Direction D

Minimal Floating

Status and controls as floating pills over a clean map. Maximum simplicity. The interface is barely there — just the map and a few floating indicators. iOS-inspired minimalism.

Direction D Notes

  • Status items as individual floating pills — maximum map visibility
  • Control buttons as floating circles — icon-only, minimal footprint
  • Cleanest visual design — the least amount of "app" visible
  • Risk: less discoverable for new users (family members) — icons without labels
  • Risk: floating elements may feel disconnected from the content
  • Best for: experienced users who want maximum immersion and minimum UI
Direction E

Desktop — Side Panel Layout

Desktop and tablet layout with journal/archive as a persistent side panel. The hex grid takes 70% of the screen; the journal and game details live alongside it. The writing session layout.

  • On phone, this collapses to Direction A/B/C layout with journal as slide-up overlay

Direction F Notes

  • Understated game-over — no fireworks, no dramatic animations
  • Thumbnail of the final map state as a visual memory of the campaign
  • Victory: warm gold title. Defeat: muted tone, same layout, same dignity
  • Four key stats: turns played, forts captured, Dark Force count, journal entries
  • Two clear actions: start a new campaign or view the archive
  • "Closing a chapter in a book" — quiet satisfaction
Direction G

Direction G Notes

  • Each campaign is a card with map name, outcome (victory/defeat), date, turns, and journal count
  • Victory in green, defeat in muted red — not dramatic, just factual
  • Simple bottom navigation: Play (active game or new), Archive, Settings
  • Tapping a card opens the game detail view (final map state + journal entries)
  • Future: map thumbnails could be added to each card for visual memory
Direction H

Journal — Quick Capture Close-Up

The journal panel in action. Open fast, dictate, close. The map stays visible. Two interaction styles: quick slide-up on phone, persistent side panel on desktop.

Direction H Notes

  • Journal slides up from bottom, taking ~50% of screen height
  • Map compresses but stays visible above — player can reference game state while writing
  • Textarea is immediately focused and ready for dictation input
  • "Save Entry" button — one tap to save, panel stays open for review
  • Previous entries visible below the input area for context recovery
  • Close button (X) dismisses journal — map returns to full size
  • On desktop: this becomes the persistent side panel (Direction E)

End of Design Directions

Review each direction and note which elements resonate. Directions can be mixed and matched.
